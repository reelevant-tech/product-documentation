## Overview

The File Upload source lets you import data by uploading a file directly from your computer. The platform parses the file and extracts fields for [mapping](/platform-guide/datahub/field-mapping).

## Configuration

| Field | Description |
|-------|-------------|
| **File upload** | Click or drag a file to the upload area. |

## Supported File Formats

The platform automatically detects the file format. The following formats are supported:

| Format | Extension(s) | Description |
|--------|-------------|-------------|
| **CSV** | `.csv`, `.tsv`, `.txt` | Comma-separated values. Delimiter is auto-detected (comma, semicolon, tab, pipe). |
| **JSON** | `.json` | Standard JSON files with a root array or object. |
| **NDJSON** | `.ndjson`, `.jsonl` | Newline-delimited JSON (one JSON object per line). |
| **XML** | `.xml` | XML files — the root element path is auto-detected. |
| **Parquet** | `.parquet` | Apache Parquet columnar format. |
| **Avro** | `.avro` | Apache Avro serialization format. |
| **XLSX** | `.xlsx` | Microsoft Excel files. |

<Info>
  Compressed files (`.gz`, `.zip`) are automatically decompressed before parsing.
</Info>

## How It Works

1. Upload your data file using the drag-and-drop area or file browser.
2. The platform detects the file format and text encoding automatically.
3. The file is stored securely and parsed.
4. Fields are extracted and made available for [mapping](/platform-guide/datahub/field-mapping).

## Updating Data

You can update the data in a file-upload datasource in two ways:

- **Re-upload via the platform** — Edit the datasource and upload a new file.
- **Push via SFTP** — When a file-upload datasource is created, Reelevant generates SFTP credentials. Your systems can push updated files to the SFTP endpoint, and the datasource will automatically re-process the latest file.

<Info>
  File upload creates an initial import. To keep data updated automatically on a schedule, consider using a [URL](/platform-guide/datahub/source-types/url), [FTP/SFTP](/platform-guide/datahub/source-types/ftp-sftp), or cloud storage source instead.
</Info>
