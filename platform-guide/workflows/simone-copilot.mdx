## What is Simone?

Simone is Reelevant's **AI copilot** for workflows. It is an intelligent assistant that helps you build, modify, and debug workflows through a natural language chat interface. Instead of manually dragging nodes and configuring each one, you can describe what you want and Simone will make the changes for you.

<Info>
  Simone is currently available to select accounts. If you don't see the AI Copilot button in your toolbar, contact your account manager.
</Info>

## Accessing Simone

1. Open any workflow in the editor.
2. Click the **AI Copilot** button (robot icon) in the bottom toolbar.
3. A chat panel opens on the right side of the editor.

## What Simone Can Do

### Build Workflows
Describe the workflow you want to create in plain language, and Simone will:
- Add entrypoint (channel) nodes.
- Create and configure data nodes with the right datasources.
- Set up logic nodes with conditions.
- Connect nodes and create branches.
- Add output nodes with content templates.

**Example prompt:** _"Create an email workflow that shows product recommendations from my product datasource, with a condition based on the user's country."_

### Modify Existing Workflows
Ask Simone to make changes to an already-built workflow:
- Add new nodes to existing branches.
- Update node configurations.
- Connect or disconnect nodes.
- Rename branches.
- Remove nodes or entrypoints.

**Example prompt:** _"Add a weather data node before the condition and create a new branch for rainy weather."_

### Review Workflows
Ask Simone to explain or review your workflow:
- Get a summary of what the workflow does.
- Identify potential issues or improvements.
- Understand complex branch logic.

### Debug Workflows
When a workflow isn't working as expected, Simone can help diagnose issues:
- Analyze error messages and suggest fixes.
- Identify misconfigured nodes.
- Apply fixes automatically when possible.

## How Simone Works

### Chat Interface
The chat panel works like a messaging interface:
1. Type your request in the input field at the bottom.
2. Simone processes your request and shows a **plan** of the actions it will take.
3. Review the plan — each step is clearly described.
4. Simone executes the actions, and you see the changes appear on the canvas in real time.
5. If something isn't right, you can ask Simone to adjust or undo.

### Plans and Actions
For complex requests, Simone creates a multi-step plan. Each step in the plan shows:
- **What will be done** — A description of the action.
- **Progress** — Whether the step is pending, in progress, or completed.
- **Result** — The outcome of each step.

### Clarification Requests
If your request is ambiguous, Simone will ask for clarification before proceeding. You can respond with additional context to help it understand your intent.

### Feedback
After Simone completes an action, you can provide feedback (thumbs up/down) to help improve its accuracy over time.

## Tips

- **Be specific** — The more detail you provide, the better the result. Mention specific datasource names, field names, and conditions.
- **Iterate** — Start with a simple request and refine. Simone remembers the conversation context.
- **Review before publishing** — Always review Simone's changes before publishing the workflow. The AI is a powerful tool but you should verify the logic.
- **Use undo** — If Simone makes an unwanted change, use the Undo button in the toolbar to revert it.
- **Combine with manual editing** — Use Simone for the heavy lifting (creating complex branch structures) and fine-tune manually.
