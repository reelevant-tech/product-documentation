## Overview

Real-time testing nodes enable automated traffic distribution that optimizes over time. Unlike a static A/B split, real-time testing can automatically adjust traffic allocation based on performance metrics.

## Configuration

Similar to [Split](/platform-guide/workflows/logic-nodes/split), but with additional settings for:
- **Testing duration** — How long the test should run.
- **Optimization metric** — What metric to optimize for (clicks, conversions, etc.).
- **Confidence level** — Statistical significance threshold.

## How It Works

1. Initially, traffic is distributed evenly across all variants.
2. As data is collected, the system analyzes which variant performs best.
3. Traffic is gradually shifted toward the winning variant.
4. Once the confidence level is reached, the test concludes and the winner receives all traffic.

## Best Practices

- Use RTT when you want the system to automatically find the best-performing variant.
- Set realistic testing durations — too short may not produce statistically significant results.
- Choose the right optimization metric that aligns with your campaign goals.
- Monitor the test progress through the workflow's statistics mode.
