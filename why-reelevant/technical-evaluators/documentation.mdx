## Documentation Structure

Reelevant maintains multiple documentation resources, each targeting a different audience and depth level.

| Resource | Audience | What it covers |
|----------|----------|----------------|
| **[Product Guide](/product-guide/introduction)** | Marketers, business users | Platform UI usage — Workflows, Content Editor, DataHub, Analytics, Account management |
| **[Advanced Guide](/advanced-guide/introduction)** | Data-savvy marketers, TAMs | DataHub deep dive, advanced Workflow configuration, Transformations |
| **[Developer Docs](/developer-docs/introduction)** | Software engineers | API reference, SDK integration (web, mobile), authentication, OpenAPI specs |
| **[Product Adoption](/product-adoption/introduction)** | Customers measuring ROI | Incremental Value methodology, Use Case Explorer, statistical model |
| **[Why Reelevant](/why-reelevant/introduction)** | Prospects (business + tech) | Capabilities, architecture, security, integration effort |

## When to Use Which Documentation

| I want to… | Go to |
|------------|-------|
| Build a Workflow and publish Content | [Product Guide → Workflows](/product-guide/workflows/overview) |
| Configure a Datasource with field mapping | [Advanced Guide → DataHub](/advanced-guide/datahub/field-mapping) |
| Integrate the web SDK in my Next.js app | [Developer Docs → Web Integration](/developer-docs/web-integration/server-side-sdk/nextjs) |
| Understand how Incremental Value is measured | [Product Adoption → Methodology](/product-adoption/incremental-value/methodology) |
| Evaluate architecture and security for procurement | [Why Reelevant → Technical Evaluators](/why-reelevant/technical-evaluators/overview) |
| Check current platform status | [status.reelevant.com](https://status.reelevant.com) |
| Request a SOC 2 report | [trust.reelevant.com](https://trust.reelevant.com) |
