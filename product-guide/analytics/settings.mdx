The Analytics Settings page lets administrators configure attribution and custom tags for consistent [Branch](/glossary) qualification and analysis.

![Analytics settings page showing attribution period configuration](/images/analytics/settings-v2.png)

Open **Analytics > Settings**, then select **General** or **Tags**.

## General settings

### Attribution period

The attribution period defines how long a conversion can be credited to a [Workflow](/glossary) interaction.

When someone clicks Reelevant [Content](/glossary) and purchases within this period, Analytics credits the conversion to that interaction.

| Field | Description |
|-------|-------------|
| **Attribution period (hours)** | The lookback window in hours (e.g., 720 for 30 days). Leave empty to use the default attribution period. |

#### Configure the attribution period

1. Navigate to **Analytics > Settings**.
2. Select **General**.
3. Enter the desired number of hours in the **Attribution period (hours)** field.
4. Click **Save**.

<Info>
  Changing this period affects the [Conversions dashboard](/product-guide/analytics/dashboards/conversions) and [Data Exploration](/product-guide/analytics/data-exploration). A longer period may include less directly related purchases.
</Info>

## Tags

Tags give each Branch controlled labels that your team can use consistently across Workflows and Analytics.

![Analytics Settings Tags tab showing tag keys, cardinality badges, and values](/images/analytics/tag-settings.png)

Each tag contains a **tag key** (the label to organise) and a list of **values**. For example, `Campaign objective` can contain `Acquisition` and `Retention`.

Only custom tags appear in this tab. Reelevant-managed tags are not editable.

### Understand cardinality

Cardinality controls how many values from one tag key can be assigned to the same Branch. It does not limit how many values you create.

![New tag key drawer showing the Single and Multiple cardinality choices](/images/analytics/tag-cardinality.png)

| Cardinality | Behaviour in a Workflow | Example |
|---|---|---|
| **Single** | Each final Branch resolves to one value from this tag key. | If several values occur along one Branch, the most downstream value is used. |
| **Multiple** | Each final Branch keeps every value assigned along it. | `Audience context: Birthday` and `Audience context: Loyalty` can both describe the same Branch. |

**Most downstream** means closest to the [Output Node](/glossary).

Imagine one Workflow qualifies `Campaign objective: Acquisition` before a second Logic Node.

A later qualification on the same final Branch assigns `Campaign objective: Retention`. Analytics uses `Retention` because it is closer to the Output Node.

With **Multiple** cardinality, both values remain attached to the final Branch.

<Tip>
  Choose **Single** when one downstream value should replace earlier values. Choose **Multiple** when every matching value should remain available.
</Tip>

### Create a tag key

1. Navigate to **Analytics > Settings**.
2. Select **Tags**.
3. Click **Add tag**.
4. Enter a unique **Name**.
5. Add an optional **Description** to help teammates use the tag correctly.
6. Choose **Single** or **Multiple** under **Cardinality**.
7. Click **Create tag key**.

Tag key names must be unique. The platform ignores capitalisation and leading or trailing spaces when checking names.

### Add values

1. Find the tag key.
2. Click the plus icon beside **Values**.
3. Enter a unique value name and an optional description.
4. Click **Add value**.

Value names must be unique within their tag key. You can expand **Values** to review every available choice.

The platform also ignores capitalisation and leading or trailing spaces when checking value names.

### Edit or delete tags

Use the pencil icon to change a tag key, its description, or its cardinality.

Expand **Values**, then use each value's action icons to edit or delete it.

<Warning>
  Deleting a tag key also deletes all its values. Deleted tag keys and values cannot be restored.
</Warning>

### Use tags in Workflows and Analytics

Created tags become available when you [qualify a Branch](/product-guide/workflows/branches). Users select controlled values but cannot create new ones there.

In Analytics, use tag keys and values to filter results or compare performance across your qualification choices.
