## Structure de la documentation

Reelevant maintient plusieurs ressources de documentation, chacune ciblant une audience et un niveau de profondeur différents.

| Ressource | Audience | Ce qu'elle couvre |
|-----------|----------|------------------|
| **[Guide produit](/fr/product-guide/introduction)** | Marketeurs, utilisateurs métier | Utilisation de l'interface — Workflows, Content Editor, DataHub, Analytics, gestion de compte |
| **[Guide avancé](/fr/advanced-guide/introduction)** | Marketeurs data-savvy, TAMs | Approfondissement du DataHub, configuration avancée des Workflows, Transformations |
| **[Documentation développeur](/developer-docs/introduction)** | Ingénieurs logiciels | Référence API, intégration SDK (web, mobile), authentification, spécifications OpenAPI |
| **[Pourquoi Reelevant](/fr/why-reelevant/introduction)** | Prospects (métier + technique) | Capacités, architecture, sécurité, effort d'intégration |
