## Présentation

Lorsque vous utilisez des données dans les applications Reelevant, vous pouvez rencontrer un cas où la façon dont elles sont stockées dans le DataHub et la façon dont vous souhaitez les utiliser dans le Workflow ou les afficher dans l'application Content ne sont pas les mêmes.

Pour vous permettre de réaliser votre cas d'usage, nous avons créé un ensemble de « transformations » qui vous permettent de transformer les données provenant du DataHub selon vos besoins.

## Content Editor

Avant de vous plonger dans l'application Content, sachez que nous proposons une liste de transformations préconfigurées dans la section « Formatters » du composant Texte. Cependant, elles sont principalement limitées aux transformations liées à l'affichage de nombres et de dates. Pour aller plus loin, vous devrez utiliser notre système de transformations en sélectionnant le sélecteur « custom » lors de l'ajout d'une nouvelle donnée :

<Note>
  Juste après, il vous sera demandé de définir un nom pour cette variable personnalisée afin de pouvoir la reconnaître, puis un éditeur de type code vous sera présenté.
</Note>

## Workflow Editor

À l'inverse, les transformations dans le Workflow se font avec le Node « Transformation » dédié, que vous pouvez ajouter à tout moment. En sélectionnant ce Node, vous disposerez (comme dans l'application Content) d'un éditeur de type code.
