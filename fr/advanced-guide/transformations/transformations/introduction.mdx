## Vue d'ensemble

Lorsque vous utilisez des données dans les applications Reelevant, il peut arriver que la façon dont elles sont stockées dans le Datahub et la façon dont vous souhaitez les utiliser dans le Workflow ou les afficher dans l'application Content ne soient pas les mêmes.

Pour vous permettre malgré tout de réaliser votre cas d'usage, nous avons créé une série de « transformations » qui vous permettent de transformer les données provenant du Datahub selon vos besoins.

## Content Editor

Avant de vous lancer dans l'application Content, sachez que nous proposons une liste de transformations pré-configurées dans la section « Formatters » du composant Text ; elles sont toutefois principalement limitées aux transformations liées à l'affichage de nombres et de dates. Pour aller plus loin, vous devrez utiliser notre système de transformations en sélectionnant le sélecteur « custom » lors de l'ajout d'une nouvelle donnée :

<Note>
  Juste après, nous vous demanderons de définir un nom pour cette variable personnalisée afin que vous puissiez la reconnaître, puis un éditeur de type code vous sera présenté.
</Note>

## Workflow Editor

À l'inverse, les transformations dans le Workflow se font à l'aide du Node « Transformation » dédié, que vous pouvez ajouter quand vous le souhaitez. En sélectionnant le Node, vous disposerez (comme dans l'application Content) d'un éditeur de type code.
