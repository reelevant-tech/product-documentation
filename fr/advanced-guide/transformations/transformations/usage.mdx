## Vue d'ensemble

Dans les deux cas, lorsque vous effectuez des transformations dans le Workflow ou le Content Editor, vous disposez d'un éditeur de type code et de notre sélecteur de « variables » juste à côté. La documentation suivante sur son utilisation vaut pour les deux, car ils utilisent le même système sous le capot.

## Syntaxe

Les transformations s'écrivent dans un « langage » spécifique qui ressemble à du code ; nous avons essayé de le faire ressembler au système de Microsoft Excel avec un peu plus d'informations. Notez que partout où vous voyez `<variable>` par la suite, cela désigne n'importe quelle donnée provenant du sélecteur de variables (que vous pouvez introduire en cliquant sur un champ). Voici un exemple :

```text
parse-number(<variable>)
```

Vous voyez ici la simple fonction `parse-number` qui prend simplement la variable de votre choix et la transforme en nombre. Mais d'autres transformations peuvent prendre des informations supplémentaires :

```text
divide(<variable 1>, <variable 2>)
```

Dans le cas ci-dessus, nous utilisons deux variables différentes pour calculer le résultat de leur division. Un autre type de transformation :

```text
truncate(<variable>, start=0, end=10, elipsis=false)
```

La transformation `truncate` (qui, comme son nom l'indique, tronque l'entrée à une taille spécifique) prend une variable, un début de troncature, un nombre maximal de caractères et l'indication d'ajouter ou non les points de suspension (...) à la fin.

Lorsque les paramètres ne sont pas des variables, vous pouvez également les définir à l'aide de leur nom :

```text
truncate(<variable>, 0, 10, false)
```

Comme vous pouvez le voir dans l'exemple ci-dessus, il est possible de ne pas définir le nom de l'option (elles ne sont pas au format name="value") ; toutefois, l'ensemble de la documentation et la plateforme les retraduiront en options nommées par souci de clarté.

<Info>
  **Important à noter**

  Vous pouvez « chaîner » les transformations, par exemple si vous souhaitez analyser en nombre le résultat d'une transformation truncate.
</Info>
