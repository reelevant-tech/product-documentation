## Vue d'ensemble

La page Paramètres Analytics permet de configurer l'attribution et les tags personnalisés de votre entreprise. Utilisez les tags pour qualifier les Branches et segmenter Analytics.

Accédez-y depuis le lien **Settings** dans la navigation latérale d'Analytics.

## Attribution

L'attribution détermine la part de chiffre d'affaires que vos [Workflows](/fr/product-guide/workflows/overview) Reelevant ont aidé à générer. Vous y choisissez le modèle, les fenêtres rétrospectives et la source de données. Voir [Attribution](/fr/product-guide/analytics/attribution) pour une explication complète.

### Modèle d'attribution

Le modèle décide comment le crédit est partagé entre les interactions d'un parcours client.

| Modèle | Comment il partage le crédit |
|-------|-------------|
| **Position-based** (par défaut) | La première et la dernière interaction reçoivent l'essentiel du crédit ; celles du milieu se partagent le reste. |
| **Linear** | Chaque interaction reçoit une part égale. |
| **Last-touch** | La dernière interaction avant l'achat reçoit tout le crédit. |

### Fenêtres rétrospectives

La fenêtre rétrospective est la durée précédant un achat pendant laquelle une interaction peut recevoir du crédit. Vous pouvez définir une fenêtre différente pour chaque type d'interaction.

| Champ | Description |
|-------|-------------|
| **Click window (hours)** | Fenêtre rétrospective pour les clics sur du Content Reelevant (par exemple, 720 pour 30 jours). Laissez vide pour la valeur par défaut. |
| **View-through window (hours)** | Fenêtre rétrospective pour les vues display (par exemple, 168 pour 7 jours). Laissez vide pour la valeur par défaut. |

### Configurer l'attribution

1. Rendez-vous dans **Analytics > Settings**.
2. Choisissez un **modèle d'attribution**.
3. Définissez les **fenêtres rétrospectives** pour chaque type d'interaction.
4. Choisissez votre **source de données** (web tracking ou customer purchases).
5. Cliquez sur **Save**.

<Info>
  Modifier l'attribution affecte la façon dont les conversions sont comptabilisées dans le [tableau de bord Conversions](/fr/product-guide/analytics/dashboards/conversions) et [Data Exploration](/fr/product-guide/analytics/data-exploration). Une fenêtre plus longue capte davantage de conversions mais peut inclure des achats moins directement attribuables.
</Info>

## Tags

Les tags donnent à chaque Branch des libellés contrôlés que votre équipe utilise de manière cohérente dans les Workflows et Analytics.

Chaque tag contient une **clé de tag** (le libellé à organiser) et une liste de **valeurs**. Par exemple, `Objectif de campagne` peut contenir `Acquisition` et `Fidélisation`.

Seuls les tags personnalisés apparaissent dans cet onglet. Les tags gérés par Reelevant ne peuvent pas être modifiés.

### Comprendre la cardinalité

La cardinalité contrôle le nombre de valeurs d'une clé de tag attribuables à une même Branch. Elle ne limite pas le nombre de valeurs créées.

![Panneau de création d'une clé de tag présentant les cardinalités Unique et Multiple](/images/analytics/tag-cardinality.png)

| Cardinalité | Comportement dans un Workflow | Exemple |
|---|---|---|
| **Unique** | Chaque Branch finale retient une valeur de cette clé de tag. | `Objectif de campagne : Fidélisation` ne peut pas être associé à `Objectif de campagne : Acquisition` sur la même Branch. |
| **Multiple** | Chaque Branch finale conserve toutes les valeurs attribuées. | `Contexte d'audience : Anniversaire` et `Contexte d'audience : Fidélité` peuvent décrire la même Branch. |

**La plus en aval** désigne la qualification la plus proche de l'[Output Node](/fr/glossary).

Imaginez un Workflow qui attribue `Objectif de campagne : Acquisition` avant un second Logic Node.

Une qualification ultérieure sur la même Branch finale attribue `Objectif de campagne : Fidélisation`. Analytics utilise `Fidélisation`, car cette valeur est plus proche de l'Output Node.

Avec la cardinalité **Multiple**, les deux valeurs restent attribuées à la Branch finale.

<Tip>
  Choisissez **Unique** lorsqu'une valeur en aval doit remplacer les précédentes. Choisissez **Multiple** pour conserver toutes les valeurs rencontrées.
</Tip>

### Créer une clé de tag

1. Rendez-vous dans **Analytics > Settings**.
2. Sélectionnez **Tags**.
3. Cliquez sur **Ajouter un tag**.
4. Saisissez un **Nom** unique.
5. Ajoutez une **Description** facultative pour guider vos coéquipiers.
6. Choisissez **Unique** ou **Multiple** sous **Cardinalité**.
7. Cliquez sur **Créer la clé de tag**.

Les noms des clés de tag doivent être uniques. La plateforme ignore la casse et les espaces placés au début ou à la fin.

### Ajouter des valeurs

1. Trouvez la clé de tag.
2. Cliquez sur l'icône plus à côté de **Valeurs**.
3. Saisissez un nom de valeur unique et une description facultative.
4. Cliquez sur **Ajouter une valeur**.

Les noms des valeurs doivent être uniques dans leur clé. Développez **Valeurs** pour consulter tous les choix disponibles.

La plateforme ignore aussi la casse et les espaces placés au début ou à la fin des valeurs.

### Modifier ou supprimer des tags

Utilisez l'icône crayon pour modifier une clé de tag, sa description ou sa cardinalité.

Développez **Valeurs**, puis utilisez les icônes de chaque valeur pour la modifier ou la supprimer.

<Warning>
  Supprimer une clé de tag supprime également toutes ses valeurs. Les clés et valeurs supprimées ne peuvent pas être restaurées.
</Warning>

### Utiliser les tags dans les Workflows et Analytics

Les tags créés deviennent disponibles lorsque vous [qualifiez une Branch](/fr/product-guide/workflows/branches). Les utilisateurs sélectionnent les valeurs contrôlées sans en créer de nouvelles.

Dans Analytics, utilisez les clés et valeurs pour filtrer les résultats ou comparer les performances de vos qualifications.
