## Vue d'ensemble

Le Node URL Parameter capture une valeur transmise via l'URL lors de l'exécution du Workflow. Cela est utile pour transmettre du contexte depuis votre ESP (fournisseur de service e-mail), votre site web ou toute intégration qui déclenche le Workflow via URL.

**Configuration :**
- **Parameter name** — Le paramètre d'URL à lire (ex. `product_id`, `campaign_date`).
- **Date format** — Si le paramètre contient une date, spécifiez le format attendu pour un bon parsing.

## Cas d'usage courants

- Transmettre des **identifiants utilisateur** depuis votre fournisseur de service e-mail pour personnaliser le contenu par destinataire.
- Transmettre des **identifiants de campagne** pour adapter le contenu au contexte de la campagne.
- Transmettre des **identifiants produit** ou des **noms de catégorie** depuis votre site web pour afficher des recommandations pertinentes.

<Tip>
  Les paramètres d'URL sont couramment utilisés pour transmettre des identifiants utilisateur, des identifiants de campagne ou des données contextuelles depuis vos outils marketing vers les Workflows Reelevant. En savoir plus sur leur configuration lors de l'[intégration](/fr/product-guide/workflows/integration).
</Tip>

## Bonnes pratiques

- Utilisez les [Variable Profiles](/fr/product-guide/workflows/settings) pour préconfigurer des modèles de paramètres d'URL pour différentes intégrations.
- Documentez toujours les noms et formats de paramètres attendus dans la description de votre Workflow.
- Envisagez d'utiliser un format de date lors de la transmission de dates pour garantir un bon parsing quelle que soit la locale.
