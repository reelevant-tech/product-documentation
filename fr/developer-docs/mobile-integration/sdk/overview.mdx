## Vue d'ensemble

Les Reelevant Mobile SDK fournissent une API unifiée à la fois pour l'**analytics** (tracking d'événements) et la **personnalisation** (récupération de contenu) dans vos applications mobiles natives. Une seule instance du SDK gère la gestion de l'identité, la collecte d'événements et le contenu personnalisé en temps réel — aucun client séparé nécessaire.

## Fonctionnalités

| Fonctionnalité | Description |
|---------|-------------|
| **Identité unifiée** | `setUser()` partage l'identité entre les événements d'analytics et les appels de personnalisation |
| **userId résolu automatiquement** | `run()` se résout automatiquement à partir de l'identité stockée — inutile de le transmettre manuellement |
| **Réponses typées** | Types de contenu discriminés (JSON, HTML, Image, Empty) avec métadonnées typées |
| **Stratégies de repli** | Choisissez entre résultats vides, propagation d'erreur ou handlers personnalisés en cas d'échec |
| **Tracking des clics** | `trackClick()` intégré sur chaque résultat pour l'attribution |
| **Exécutions concurrentes** | `runAll()` exécute plusieurs Workflows en parallèle |
