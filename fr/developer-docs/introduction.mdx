## Vue d'ensemble

Reelevant propose plusieurs modes d'intégration selon votre Channel et votre architecture :

<CardGroup cols={2}>
  <Card title="Intégration Web" icon="globe" href="/fr/developer-docs/web-integration/overview">
    SDK côté serveur pour Next.js, Nuxt, Remix, SvelteKit — ou le script de tracking côté client.
  </Card>
  <Card title="Intégration mobile" icon="mobile" href="/fr/developer-docs/mobile-integration/overview">
    SDK natifs pour Android (Kotlin), iOS (Swift) et Flutter (Dart).
  </Card>
  <Card title="Référence API" icon="code" href="/fr/developer-docs/api-reference/introduction">
    API REST pour les Datasources, les Contents, les Workflows et le Collector d'événements.
  </Card>
  <Card title="Guides" icon="book" href="/fr/developer-docs/guides/browser-extension-integration">
    Intégration de l'extension de navigateur, dépannage et patterns avancés.
  </Card>
</CardGroup>

## Authentification

Tous les endpoints de l'API nécessitent une authentification OAuth2. Consultez le [guide d'authentification](/fr/developer-docs/api-reference/authentication) pour savoir comment obtenir et utiliser des tokens d'accès.
