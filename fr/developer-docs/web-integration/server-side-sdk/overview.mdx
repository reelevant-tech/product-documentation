## Qu'est-ce que le Server Side SDK ?

Le Reelevant Server Side SDK (`@rlvt/web-sdk`) vous permet de récupérer du contenu personnalisé **sur le serveur** — pendant le rendu de la page — afin qu'il arrive dans le payload HTML initial. Les visiteurs voient immédiatement le bon contenu, les moteurs de recherche l'indexent et il n'y a aucun décalage de mise en page visible.

## Comment ça marche

<Steps>
  <Step title="Installer le SDK">
    Ajoutez le package `@rlvt/web-sdk` à votre projet.
  </Step>
  <Step title="Créer un client">
    Initialisez un `ReelevantClient` — aucune configuration n'est requise (le SDK utilise le Runner de production par défaut).
  </Step>
  <Step title="Appeler le Runner">
    Dans votre route ou votre composant serveur, appelez `client.run()` avec l'ID du Workflow et l'entrypoint. Le SDK appelle le Runner Reelevant et renvoie du contenu typé (HTML, JSON ou image).
  </Step>
  <Step title="Effectuer le rendu du contenu">
    Injectez le contenu renvoyé dans le template de votre page. Le SDK marque la sortie avec `data-rlvt-ssr` afin que le tracker côté client existant sache qu'il doit ignorer ces zones.
  </Step>
  <Step title="Suivre les clics">
    Après l'affichage du contenu, configurez toujours le tracking des clics — utilisez soit `redirectionUrl` comme `<a href>`, soit appelez `result.trackClick()` côté serveur. Cela garantit que les données d'attribution sont capturées pour chaque interaction.
  </Step>
</Steps>
